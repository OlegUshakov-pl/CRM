# CRM Project - Bug Report & Fix Plan

## CRITICAL BUGS

### 1. ProjectFileSystemStorage._open() — вызывает NotImplementedError
- Файл: projects/utils.py:89-90
- Описание: super(FileSystemStorage, self)._open(name, mode) вызывает Storage._open() (абстрактный), а не FileSystemStorage._open()
- Исправление: заменить на super()._open(name, mode)

### 2. Open Redirect в task_delete
- Файл: tasks/views.py:84-86
- Описание: параметр next не валидируется, возможен редирект на вредоносный сайт
- Исправление: добавить проверку url_has_allowed_host_and_scheme

### 3. Race Condition в generate_unique_slug
- Файл: core/models.py:88-91
- Описание: select_for_update() снимает блокировку до save(), возможен IntegrityError
- Исправление: обернуть в цикл с retry при IntegrityError

### 4. File Descriptor Leak / Memory Bloat в project_export
- Файл: projects/views.py:230-236
- Описание: f.read() загружает весь zip в память; файл закрывается до FileResponse
- Исправление: передать открытый файловый объект напрямую в FileResponse

## HIGH PRIORITY

### 5. Dashboard загружает ВСЕ задачи без лимита
- Файл: core/views.py:33
- Исправление: добавить [:10] или другой разумный лимит

### 6. Контекстный процессор загружает все проекты на каждую страницу

### 7-13. N+1 запросы в contacts, tasks, notes, documents, materials, companies
- Файлы: contacts/views.py, tasks/views.py, notes/views.py, documents/views.py, materials/views.py, companies/views.py
- Исправление: добавить select_related/prefetch_related

### 14. Нет проверки прав доступа к объектам

### 15. API ключи в URL (Google provider)

### 16. Неправильные URL в AI handlers (id вместо slug)

### 17. Template path generator использует generators/
