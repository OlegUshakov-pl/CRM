Сценарий создания современной и красивой CRM на Django 6.0.5 + Tailwind CSS
Название проекта: CRM
Версия Django: 6.0.5
Python: 3.14+
База данных: SQLite (для разработки)
Frontend: Tailwind CSS + Alpine.js + HTMX
Иконки: Lucide
Шрифт: Montserrat
Язык: English

Этап 0: Подготовка и планирование

Определить цели CRM и основные сущности.
Утвердить ключевой UX-приём: добавление всех записей через боковое полупрозрачное меню слева.
Создать репозиторий и настроить виртуальное окружение.


Этап 1: Инициализация проекта и базовая настройка

Создать проект и необходимые приложения:
core — базовые шаблоны, sidebar, dashboard
accounts — авторизация и профиль
contacts — Company и Contact
projects — Project, Material, Task
notes — Notes
tasks — Tasks

Настроить settings.py, Tailwind CSS и django-browser-reload.
Создать основной шаблон base.html с боковой навигацией.


Этап 2: Дизайн и UI/UX структура

Разработать современный дизайн с поддержкой светлой и тёмной темы.
Создать постоянную боковую панель (Sidebar) слева.
Главное требование: Добавление новых записей осуществляется через боковое полупрозрачное меню (slide-over), выезжающее слева.
Верхняя панель (Topbar) с глобальным поиском и профилем.
На каждой странице раздела (Projects, Contacts, Companies и т.д.) должна присутствовать заметная кнопка «+ Добавить» (в правом верхнем углу страницы).
При нажатии на кнопку «+ Добавить» открывается полупрозрачное боковое меню слева для создания новой записи.
Все формы создания и редактирования будут использовать одно и то же боковое меню (с динамическим содержимым через HTMX).


Этап 3: Создание моделей данных

Разработать модели в соответствующих приложениях:
contacts: Company, Contact
projects: Project, Material, Task
notes: Note

Добавить стандартные поля (created_at, updated_at, created_by, is_active).
Настроить все необходимые связи между моделями.
Применить миграции.


Этап 4: Главная страница — Dashboard

Красивый дашборд с карточками статистики.
Кнопка «+ Создать проект» (открывает боковое меню слева).
Блоки: Текущие проекты, Задачи на сегодня, Недавняя активность.
Переключение видов отображения проектов (Grid / List).


Этап 5: Авторизация и безопасность

Реализовать вход, выход и профиль пользователя.
Защитить все страницы CRM.


Этап 6: CRUD-функционал с боковым меню
Для всех основных разделов (Projects, Companies, Contacts, Tasks, Notes, Materials) реализовать следующее:

На каждой странице раздела в правом верхнем углу размещается кнопка «+ Добавить».
При нажатии на кнопку открывается полупрозрачное боковое меню слева (slide-over panel) с формой создания новой записи.
Редактирование записи также открывается через то же боковое меню слева (по кнопке "Изменить" на карточке или в таблице).
Просмотр детальной информации — в основном контенте или в отдельном модальном окне (по желанию).
Удаление — через подтверждение (диалоговое окно).
Поддержка фильтров, поиска и пагинации на каждой странице списка.

Важное правило:
Вся работа с созданием и редактированием записей происходит только через боковое полупрозрачное меню, выезжающее слева.

Этап 7: Дополнительный функционал

Полная связь между сущностями (Проект ↔ Компания ↔ Контакт).
Статусы проектов и задач.
Загрузка изображений для проектов и контактов.
Система заметок и задач.
Лента недавней активности.
Фильтры и расширенный поиск.


Этап 8: Полировка интерфейса и UX

Сделать плавную анимацию открытия/закрытия бокового меню.
Обеспечить адаптивность (на мобильных устройствах меню должно работать корректно).
Добавить красивые пустые состояния (empty states).
Оптимизировать производительность запросов.
Финальная шлифовка дизайна.


Этап 9: Финализация проекта

Настроить все маршруты и представления.
Провести полное тестирование.
Подготовить проект к запуску.
Создать подробный README.md.
