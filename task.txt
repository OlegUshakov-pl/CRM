ТЗ: Внедрение AI-ассистента в CRM
Версия: 1.0
Стек: Django + Tailwind CSS + HTML
Цель: Встроить в текущую CRM чат-ассистента на базе локальной LLM. Ассистент управляет данными CRM, работает с внешними сайтами и файлами. В v2 добавить голос.

Текущая структура CRM: Dashboard, Projects, Materials, Drawings, Tasks, Documents, Companies, Contacts, Notes, Import Project.

сначала создать новый app назвать assistant

1. Глобальные требования
1.1 Безопасность: AI выполняет действия только от имени авторизованного пользователя CRM и в рамках его прав. Все действия AI логируются в "Журнал событий" с пометкой "AI Chat".
1.2 Отмена действий: Для операций создания/изменения данных в чате выводить кнопку "Отменить" в течение 10 секунд.
1.3 Подтверждение: Перед выполнением критичных действий AI обязан запросить подтверждение: "Создать проект 'Название' на дату X. Подтверждаете?"
1.4 Производительность: Ответ AI на простую команду ≤ 3 сек. На сложную типа парсинга сайта ≤ 15 сек с промежуточным статусом "Ищу заголовки...".

2. Модуль 1: AI Chat - ядро системы
2.1 Размещение: Добавить плавающую иконку чата в правом нижнем углу CRM. Иконка доступна на всех страницах.
2.2 Интерфейс панели чата:

Открывается поверх контента, ширина 450px на десктопе, 100% на мобильном.
Режимы: "Обычный" 450px и "Широкий" 50% экрана. Переключение по кнопке.
Элементы: история сообщений, поле ввода "Что сделать?", кнопка микрофона "Скоро", кнопка "Очистить", кнопка "Развернуть".
Сообщения AI: текст + интерактивные блоки. Например, после "создай проект" блок с данными проекта и кнопками "Открыть", "Отменить".
2.3 Базовые возможности AI без действий:

Отвечать на вопросы по данным CRM: "Сколько у меня активных задач?", "Покажи последний документ".
Объяснять как пользоваться CRM: "Как добавить материал?".
3. Модуль 2: Управление CRM через чат

Логика работы:

AI парсит команду → определяет нужную функцию и параметры.
Если параметров не хватает → задает уточняющий вопрос.
Выполняет действие → возвращает результат в чат с кнопками.
4. Модуль 3: Окно браузера для работы с интернетом
Задача: Дать AI "глаза", чтобы он мог открывать сайты по команде и доставать информацию.

4.1 Активация: Когда AI выполняет команду связанную с сайтом, панель чата переходит в "Широкий" режим. Правая половина становится "Окном браузера".
4.2 Функции "Окна браузера":

Отображает скриншот страницы, которую AI сейчас анализирует.
Не интерактивно. Пользователь не может кликать. Только просмотр.
Кнопки над окном: "Открыть в новой вкладке", "Скачать скриншот", "Закрыть окно".
4.3 Команды для AI:
"Открой bbc.com и выведи 5 главных заголовков" → AI делает скрин + парсит заголовки в чат.
"Найди на сайте [URL] все ссылки на PDF" → AI парсит страницу, отдает список ссылок в чат.
"Покажи мне сайт [URL]" → AI просто открывает скрин в окне браузера.
4.4 Ограничения: Запретить открывать сайты из черного списка. Таймаут на загрузку 20 сек.

5. Модуль 4: Песочница "AI Files"
Задача: Безопасное хранилище для всего, что AI скачал из интернета.

5.1 Новый раздел меню: Добавить пункт "AI Files" в левое меню CRM, под "Notes". Иконка: папка со звездой.
5.2 Интерфейс раздела:

Таблица: Превью, Имя файла, Источник URL, Дата загрузки, Размер, Действия.
Действия для каждого файла: "Скачать на ПК", "Прикрепить к Проекту", "Удалить".
Массовые действия: "Удалить выбранные", "Очистить файлы старше 30 дней".
Фильтры: Картинки, PDF, Документы, Все.
Счетчик занятого места: "Занято 120 МБ из 1 ГБ".
5.3 Интеграция с чатом:
Команда "Скачай картинку с [URL]" → AI скачивает, кладет в AI Files, в чате пишет "Готово. Файл logo.png в AI Files. Прикрепить к проекту?"
Команда "Прикрепи последний скачанный PDF к проекту [Имя]" → AI берет последний файл из AI Files и крепит в Project Documents.
5.4 Правила безопасности:
Разрешены только: jpg, png, pdf, docx, xlsx, dwg.
Запрещены: exe, zip, файлы > 50 МБ.
Автоудаление всех файлов старше 30 дней. Раз в сутки.
6. Модуль 5: Подготовка к голосу v2
6.1 UI: Иконка микрофона в поле ввода чата. По клику на v1 показывает tooltip "Голосовое управление будет добавлено в следующем обновлении".
6.2 Backend: Вся логика AI должна работать через единый API-endpoint /api/ai/chat/. Чтобы в v2 можно было подать на него текст после распознавания голоса и ничего не менять.
6.3 Логирование: Сохранять все пары "запрос пользователя → ответ AI" для будущего дообучения STT-модели на командах.

7. Порядок сдачи и приемки
7.1 Этап 1: Сдать Модуль 1 + Модуль 2. Критерий: 10 тестовых команд из таблицы п.3 выполняются без ошибок.
7.2 Этап 2: Сдать Модуль 3 + Модуль 4. Критерий: команда "открой bbc.com, найди pdf, скачай его и прикрепи к проекту Test" выполняется от начала до конца.
7.3 Этап 3: Сдать Модуль 5. Критерий: иконка микрофона есть, API готово.

Что не входит в v1: Голосовой ввод/вывод, самостоятельный серфинг AI по ссылкам, генерация документов, анализ содержимого PDF/DWG.
