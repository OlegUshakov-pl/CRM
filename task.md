# Техническое задание: Развитие изолированного модуля "Library" внутри текущей CRM

## Контекст и визуальный анализ
Модуль **Library** является внутренней экосистемой CRM-системы. Текущий интерфейс содержит левое сквозное меню CRM (где активен пункт *Library*), верхнюю панель пользователя и центральную контентную зону, которая сейчас отображает только простейший список документов.

**Цель:** Превратить текущую контентную зону модуля "Library" в мощный хаб для работы с базой знаний, медиафайлами и внешними статьями, сохранив единый визуальный стиль CRM (стилистику Tailwind, шрифты, отступы и логику переходов без перезагрузки страниц через **HTMX**).

---

## Детальные требования к изменениям интерфейса CRM

### 1. Модернизация главного экрана Library (`liba_2.png`)
* **Интеграция гибридного поиска (под Breadcrumbs):**
  * Прямо под строкой `Library / All Documents` внедрить горизонтальную поисковую панель.
  * Внутри панели разместить переключатель (Dropdown или сегментированный control): **[В статьях (CRM) | Google | DuckDuckGo]**.
  * При выборе "В статьях" ввод текста запускает живой HTMX-поиск по локальной базе CRM (`hx-trigger="keyup changed delay:500ms"`). При выборе Google/DuckDuckGo поисковый запрос перенаправляет пользователя на внешнюю вкладку поисковика.
* **Переход от списка к контентной сетке (Grid):**
  * Текущую карточку `Simple` заменить на визуальную сетку карточек (CSS Grid). 
  * Если статья спарсена по URL или имеет вложение, карточка в сетке должна отображать изображение-превью, заголовок, автогенерацию выжимки (summary), теги, категорию и дату.
* **Расширение бокового фильтра (Компонент "Filters"):**
  * К существующим селекторам *Category* и *File Type* добавить блок **Tags** (облако тегов с возможностью клика) и фильтр по дате добавления. Любой клик по фильтру мгновенно обновляет сетку карточек через `hx-get`.

### 2. Новый сценарий: Экран добавления материала (`+ New Document`)
При клике на фиолетовую кнопку `+ New Document` в текущий контентный контейнер CRM загружается интерфейс создания с вкладками (Tabs):
* **Вкладка "Импорт по URL":** Инпут для ссылки, кнопка "Извлечь". При клике включается HTMX-спиннер (`hx-indicator`), бэкенд парсит страницу, вытягивает текст/картинку и создает локальную копию статьи в CRM.
* **Вкладка "Ручной редактор":** Полноценное текстовое поле (Markdown или WYSIWYG) для написания статьи с нуля.
* **Общие метаданные (справа или снизу формы):** 
  * Выбор категории (с возможностью тут же нажать на `+` и создать новую категорию в мини-модалке).
  * Поле ввода тегов (динамические input-chips).

### 3. Новый экран: Менеджер категорий (`Categories`)
* При клике на пункт меню **Categories** в левом сайдбаре CRM, центральная область обновляется.
* Отображается список всех категорий CRM-библиотеки в виде аккуратной таблицы: *Название категории | Кол-во документов | Дата создания | Действия (Редактировать / Удалить)*.
* Удаление происходит «на лету» через `hx-delete` с анимацией скрытия строки.

### 4. Новые медиа-разделы CRM
Поскольку библиотека теперь хранит не только текст, в контентную область добавляются два новых представления (переключение через боковое меню или табы):
* **Галерея изображений (Photo Gallery):** Сетка всех изображений, которые были прикреплены вручную или выдернуты парсером из веб-ссылок. При клике — просмотр во весь экран. У каждого фото есть мини-ссылка на родительский документ в CRM.
* **Каталог файлов (Files Catalog):** Раздел для загруженных документов (PDF, DOCX, ZIP и др.). Содержит зону Drag-and-Drop для быстрой загрузки файлов в CRM и таблицу со списком файлов, их размером и кнопками скачивания.

## Техническое задание для генерации кода
Реализуй набор Django-шаблонов для контентной зоны CRM, используя классы **Tailwind CSS** (в строгом соответствии с палитрой и стилем `liba_2.png`) и **HTMX** для бесшовного SPA-переключения между разделами (Главная, Создание, Категории, Галерея, Файлы) внутри главного контент-контейнера CRM.
